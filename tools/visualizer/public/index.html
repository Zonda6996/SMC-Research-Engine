Зоны ликвидности POI 2.4

—
Мои зоны ручная разметка
Справка по зонам

Зона = полка ликвидности (стек свежих пулов, порезанный по провалам плотности) + невыметенный экстремум. Прямоугольник: сплошная граница — near, пунктир — far, яркость заливки — сила стека, клик — фокус. Смерти: провал телом за far · фитиль насквозь · снято ≥ 50% стека · 300 баров без подпитки. Родство стеков: ≥ 50% общего notional = один объект; тронутая старшая главнее, нетронутую заменяет поколение, удерживающее её массу. «Заменена» ≠ «устарела»: замену видно в деталях. Правила — SPEC §16.12–§16.16.

Подтверждение 1.6 · 4h→15m

—
Справка по подтверждению

Последовательность: заход → остановка (направленное закрытие после 4 тихих баров) → отскок (6 баров) → пересвип невыметенного якоря → защита → тест слабости (объём возобновления выше отката) → вход; стоп за экстремумом свипа, тейк 2R. Попытка умирает: 96 баров бездействия · пробой закрытиями · конец окна зоны (со состоявшимся свипом — доигрывается). Вход отменяется при риске > 1.5 ATR15. Тусклые зоны в обзоре 4h — раньше 15m-истории (~208 дней): нет данных, не логики.

Heatmap ликвидаций 2.0 · OI

Справка по heatmap

Каждая свеча с объёмом выше среднего открывает позиции; уровни ликвидаций 5×–100× копятся в ценовых бинах и сливаются в полосы (крутилка склейки — в полки). Яркость/толщина — накопленная плотность; проход цены снимает ликвидность. Где Binance отдаёт OI (~30 дней) — вклад по росту открытого интереса и тейкер-сплиту, maintenance margin 0.4%. В статусе — ближайшие «магниты».
